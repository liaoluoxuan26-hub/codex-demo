# 短视频课程文案拆解器

这是一个纯前端静态网页工具，用来辅助处理已经转写成文字的短视频、直播课或课程文案。它不是 ChatGPT，也不会调用真正的大模型 API；它的定位是：文本清洗器、基础结构拆解器、ChatGPT 深度拆解提示词生成器和短视频文案处理辅助工具。

## 它能做什么

- 记录视频链接，方便保留素材来源。
- 粘贴已经提取出来的视频文字、字幕或口播稿。
- 清洗多余空格、重复标点、重复句、明显英文乱码，以及 `token`、`tokens`、`king token` 等识别噪声。
- 基于前端规则做基础拆解：课程标题、核心主题、主要观点、关键案例、金句摘录、可执行步骤和短视频口播文案。
- 优先把“比如、例如、举个例子”等案例句归入关键案例，主要观点更偏向判断句、总结句和因果句，金句摘录更偏向“不是……而是……”等有传播力的表达。
- 生成一段「复制给 ChatGPT 的深度拆解提示词」，方便用户把清洗后的原文交给 ChatGPT 做真正深度理解。
- 一键复制全部结果，也可以单独复制每个结果模块、清洗文本和 ChatGPT 提示词。
- 使用响应式布局，适配手机端访问。
- 保持 GitHub Pages 友好的静态部署结构，只需要 `index.html`、`style.css`、`script.js` 即可运行。

## 它不能做什么

- 不能直接解析抖音、快手、B站等平台的视频链接。
- 不能自动下载视频、自动提取音频或自动识别字幕。
- 不能像真正的大模型一样深度理解长课程全文。
- 不能保证所有拆解结果都达到人工编辑或 ChatGPT 深度分析的质量。

## 为什么不能直接解析抖音、快手、B站链接

抖音、快手、B站等平台的视频内容通常需要登录态、接口权限、反爬校验、音视频下载、语音识别和字幕解析能力。当前项目是部署在 GitHub Pages 上的纯前端静态网页，没有后端服务、数据库、登录系统和平台授权接口，因此不能稳定、合规地直接解析这些平台链接。

## 为什么纯前端不能达到真正 AI 理解能力

这个工具运行在浏览器里，只使用 JavaScript 规则做清洗、关键词评分、句子筛选和模板化改写。它没有调用 OpenAI、ChatGPT 或其他大模型 API，也没有后端算力支持。因此它适合做初步整理，但不能假装具备真正的语义理解、长文本推理和高质量改写能力。

## 推荐使用流程

1. 用飞鸟、斑马等工具先把短视频、直播课或课程内容提取成文字。
2. 把视频链接粘贴到「视频链接」输入框，用于记录来源。
3. 把提取出来的文字粘贴到「已提取视频文字」输入框。
4. 点击「清洗文本」，先去掉空格、乱码、重复句和识别噪声。
5. 点击「开始拆解」，获得基础结构拆解结果，并重点检查主要观点、关键案例、金句摘录和可执行步骤是否符合后续使用场景。
6. 点击「生成 ChatGPT 深度拆解提示词」。
7. 点击「复制 ChatGPT 深度拆解提示词」，发给 ChatGPT 做深度处理。
8. 根据 ChatGPT 输出继续整理成学习笔记、短视频口播稿或课程复盘材料。

## 如何通过 GitHub Pages 使用

1. 将本仓库推送到 GitHub。
2. 打开仓库的 **Settings → Pages**。
3. 在 **Build and deployment** 中选择从分支部署。
4. 选择主分支和根目录（通常是 `main` / `/root`）。
5. 保存后等待 GitHub Pages 自动构建。
6. 构建完成后，打开 Pages 提供的网址即可使用。

## 本地使用

直接用浏览器打开 `index.html` 即可。项目不需要安装依赖、不需要后端、不需要数据库。

## 文件说明

- `index.html`：页面结构、输入区、操作按钮、拆解结果区和 ChatGPT 提示词区。
- `style.css`：移动端优先的响应式样式。
- `script.js`：文本清洗、基础拆解、提示词生成和复制逻辑。
